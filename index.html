01 — concepto base

¿Qué es un
Agente de IA?

Un asistente que piensa, planifica y actúa por ti de forma autónoma. Entiende tu objetivo, arma un plan y lo ejecuta paso a paso, revisando y corrigiendo su propio trabajo hasta entregarte un resultado terminado.

No te da una idea: te trae la tarea hecha.

Motor de razonamiento y toma de decisiones
🔄

Percibir → Razonar → Actuar → Observar
🔧
Herramientas
Terminal, filesystem, compilador, browser
📋
Instrucciones
Objetivos, límites y comportamiento esperado

06 — frontmatter · cómo Copilot decide qué leer

Frontmatter siempre,
body solo si aplica

estándar abierto
Copilot · Claude · Cursor

Cada artefacto empieza con un bloque YAML que Copilot siempre lee. El body solo se carga si el frontmatter lo justifica — así tenés decenas de artefactos sin saturar el contexto.

📄 instructions · *.instructions.md
trigger: glob match
---
applyTo: 'src/**/*.tsx'
description: 'Reglas de componentes React'
excludeAgent: code-review  # opcional
---
Se carga automáticamente cuando el archivo que editás matchea el applyTo — sin invocación manual.
📦 skills · SKILL.md
trigger: match semántico
---
name: react-frontend
description: >-
  Activa al crear componentes React,
  hooks, features con Vite + TypeScript.
---
Copilot evalúa la description contra tu prompt y carga el body solo si hay match semántico.
💬 prompts · *.prompt.md
trigger: slash command
---
name: new-react-app
description: 'Crear app React desde cero'
agent: react-developer
tools: [filesystem, terminal]
---
Se carga cuando el dev escribe /new-react-app — Copilot delega al agente declarado en agent:.
🤖 agents · *.agent.md
trigger: invocación explícita
---
name: react-developer
description: >-
  Orquesta el flujo end-to-end:
  plan → confirma → construye.
tools: [filesystem, terminal, compiler]
---
Se carga cuando se invoca explícitamente (desde un prompt o el chat). Activa sus tools y el protocolo del body.
⚡ ciclo de carga · idéntico para los 4 artefactos
↳ click en cada paso
PASO 1
Lee TODO el frontmatter
de los artefactos en .github/ — barato
→
PASO 2
Evalúa el trigger
glob · semántico · slash · invocación
→
PASO 3
Carga el body
solo de los artefactos relevantes
El frontmatter es el contrato de carga — define cuándo aplica. El body es la implementación — qué hace. Fuente: docs.github.com · code.visualstudio.com

FLUJO TRADICIONAL

Reuniones de requisitos

→ Code review + QA
→ Deploy
FLUJO AGÉNTICO
⏱ minutos

→ Miguel ajusta / confirma
→ Agente construye + valida
