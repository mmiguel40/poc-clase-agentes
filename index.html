01 — concepto base

¿Qué es un
Agente de IA?

Un sistema que usa un LLM para tomar decisiones, ejecutar acciones
con herramientas y operar en un loop autónomo hasta completar
un objetivo — sin intervención humana en cada paso.

🧠
Modelo LLM
Motor de razonamiento y toma de decisiones
🔄
Loop autónomo
Percibir → Razonar → Actuar → Observar
🔧
Herramientas
Terminal, filesystem, compilador, browser
📋
Instrucciones
Objetivos, límites y comportamiento esperado

05 — skill anatomy

Qué contiene
react-frontend

El frontmatter define cuándo activarse.
El body son las reglas que el agente
debe seguir al construir el proyecto.

DESCRIPCIÓN — el trigger
Frases que activan la skill: "crea componente", "estructura React", "nuevo hook"
ESTRUCTURA
Feature-based folders. Un hook por feature. Separación lógica / UI obligatoria
REGLAS DURAS
Sin any. Sin lógica en componentes UI. Interfaces explícitas. PascalCase / use* / kebab-case
react-frontend react-state react-ui
3 skills · se activan juntas según contexto
.github/skills/react-frontend/SKILL.md
1---
2name: react-frontend
3description: >-
4Activa al crear componentes React, hooks,
5features, estructura de carpetas o apps
6con Vite + TypeScript.
7---
8
9## Estructura
10- Feature-based folders # NO type-based
11- src/features/[nombre]/
12 ├── use[Nombre].ts # lógica aquí
13 ├── [Nombre]List.tsx # solo UI
14 └── [nombre].types.ts
15## Reglas
16- Prohibido: any, var, lógica en JSX
17- localStorage: solo en services/
⚡ cómo funciona la carga lazy
↳ click en cada paso
PASO 1
Lee solo el frontmatter
name + description de todas las skills — muy liviano
→
PASO 2
Infiere relevancia
¿el contexto coincide con la description? → sí / no
→
PASO 3
Carga el body completo
solo si la skill fue marcada como relevante
El body puede ser largo y detallado — no penaliza el contexto si la skill no se activa. La description es el único texto que siempre se lee.
estándar abierto · compatible con Copilot, Claude Code, Cursor
06 — caso de uso · el reto

Distribuidora ABC
tiene un problema

👩
Luisana
PM · Distribuidora ABC
El dolor

El equipo comercial de Luisana envía decenas de cotizaciones por semana. Las trackean en una hoja de cálculo compartida — se pierden estados, se duplican filas, y nadie sabe cuáles están vencidas.

Lo que necesita
  • ✓ Tablero interno para 6 personas — ver, agregar, editar, marcar estado
  • ✓ Filtrar por cliente, estado y fecha
  • ✓ Lo necesita esta semana
  • ✗ No tiene un dev libre en el equipo
Dos formas de abordarlo
FLUJO TRADICIONAL
⏱ semanas
Reuniones de requisitos
→ Decidir arquitectura
→ Setup proyecto + tooling
→ Implementar
→ Code review + QA
→ Deploy
FLUJO AGÉNTICO
⏱ minutos
Luisana describe el reto
→ Agente razona + propone plan
→ Luisana confirma (o ajusta)
→ Agente construye + valida
→ App corre en localhost
⚡
Vamos a abordarlo con las capacidades agénticas de GitHub Copilot: Custom Agents orquestando el flujo, Skills con las reglas del proyecto, y Prompts como punto de entrada.
